How the circuit is built

The math behind the model, the exact 2025 constants, and the IRS sources every edge is lifted from.

The graph

Each line of Form 1040 and Schedules 1–3 is a node; each line label is a signed operation over earlier lines. Because every reference points to a lower-numbered line, the line numbering is a topological sort and the return evaluates in one top-to-bottom pass. The edges are:

  • Aggregation (Σ): pure sums, all parents +1 — lines 9, 14, 18, 21, 24, 25d, 32, 33.
  • Signed difference: line 11a = 9 − 10, 15 = 11b − 14, 22 = max(18 − 21, 0) — where deductions and credits enter as negative parents, several with a max(·,0) clamp.
  • One nonlinear edge: tax = brackets(taxable), a convex piecewise-linear map (the Tax Table / Tax Computation Worksheet).
  • Cross-form identity: schedule subtotals routed onto a 1040 line (coefficient 1).

The two fixed points

Two pairs of lines reference each other and cannot be ordered away. Each is solved by iterating a self-map x ← G(x) to a fixed point.

SCC #1 — IRA deduction D ↔ taxable Social Security S

The Social Security Benefits Worksheet nets the IRA deduction out of provisional income (so larger D ⇒ smaller S); the IRA deduction phases out on modified AGI, which includes S (so larger S ⇒ smaller D). The composed return map has slope

G′(S) = r_ss · r_po,   r_ss ∈ {0, ½, 0.85},  r_po = C/(hi−lo) = 0.7 (single) ⇒  κ = 0.595 < 1

so G is a contraction (Banach ⇒ unique fixed point, geometric convergence; piecewise-linear ⇒ often exact in a few steps). The 2025 active-participant phase-out band on modified AGI: $79,000–$89,000 (single/HoH), $126,000–$146,000 (MFJ, you covered), $0–$10,000 (MFS) — Pub. 590-A.

SCC #2 — SE-health deduction D ↔ premium tax credit P

The §162(l) deduction equals premiums minus the credit; the credit (Form 8962) depends on MAGI, which the deduction reduces. The return map H(D) = premiums − PTC(base − D) has slope

H′(D) = −(a + a′m),   a = applicable percentage ∈ [0, 0.085] ⇒  |κ| ≈ 0.1

an even tighter contraction, converging in ≤ 9 steps. Resolved by the iterative method of Pub. 974 (the exact simultaneous form is Rev. Proc. 2014-41).

Minimizing tax

“Least tax” is min F over the lever variables subject to the contribution caps and eligibility guards. Because the only nonlinearity is the convex bracket function, this is a small mixed-integer linear program: the graph’s affine edges become linear equality constraints, the lever inputs become continuous decision variables (the Saver’s-Credit tier is a binary), and the bracket map becomes a convex piecewise-linear relaxation. The genuinely non-convex parts — the Tax Table’s $50-row rounding and the two cyclic loops — live outside the linear program (a post-solve reconciliation, and the fixed-point solvers). The on-page calculator uses a fast grid search over the same feasible set.

Assumptions & limits

  • The main calculator assumes the standard deduction and (unless “active participant” is ticked) full IRA deductibility; it does not model itemizing, preferential capital-gains rates, the QBI deduction, AMT, or state tax.
  • The SE-health solver uses an annual, single-benchmark shortcut (not Form 8962’s month-by-month proration) and approximates the ½-SE-tax term.
  • MAGI add-backs (tax-exempt interest, excluded foreign income, nontaxable SS) are not modeled; fold them into “other income” if relevant.
